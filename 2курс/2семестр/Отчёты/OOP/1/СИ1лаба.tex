	\begin{enumerate}
		\item Выбрать и согласовать с преподавателем предметную область.

	\subsection*{Разработка демонстрационного приложения}
	\addcontentsline{toc}{subsection}{Разработка демонстрационного приложения}
	Для демонстрации созданной структуры разработано конскольное приложение.
	
	\textbf{Демонстрация инкапсуляции:}
	Данные классов (\texttt{revenue}, \texttt{purpose}) скрыты в секции \texttt{private}. Доступ к ним и изменение состояния объектов возможны только через публичные методы.
	
	\textbf{Демонстрация наследования:}
	Объекты \texttt{Commersorg} и \texttt{Noncomersorg} содержат все поля и методы класса \texttt{Organization}.При вызове метода \texttt{ARGANIZATII[2]->expandBusiness()}, который не определен в базовом классе, но успешно вызывается у объекта дочерней коммерческой организации благодаря наследованию и полиморфизму.
	
	\textbf{Демонстрация полиморфизма:} 
	Все объекты хранятся в векторе указателей на базовый класс \texttt{std::vector<Organization*>}. При вызове виртуальных методов (\texttt{payTaxes()}, \texttt{report()}) программа автоматически выполняет версию метода, соответствующую реальному классу объекта.
	
	В консоль будет выведен налог для коммерческой организации, отчет для коммерческой организации, и вызван метод \texttt{expandBusiness()}, который увеличит количество подписчиков.
