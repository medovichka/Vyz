\section*{Цель работы}
\addcontentsline{toc}{section}{Цель работы}
Изучение принципов объектно-ориентированного программирования на языке C++.

\section*{Задание}
\addcontentsline{toc}{section}{Задание}

\begin{enumerate}
	\item Выбрать и согласовать с преподавателем предметную область.
	\item Создать иерархию классов, состоящую не менее чем из одного родительского и двух дочерних классов.
	\item В каждом классе определить не менее двух членов данных, не менее двух собственных, а для дочерних - не менее двух унаследованных и двух переопределенных (перекрытых) член-функций.
	\item Разработать приложение, демонстрирующее принципы инкапсуляции, наследования и полиморфизма.
\end{enumerate}
* Работу с объектами осуществлять только через переменные родительского типа. \\
* Каждый класс в отдельном файле. \\
* Отчет должен содержать описание предметной области, схему структуры иерархии классов (можно использовать диаграмму классов), описание методов, особенности реализации.

\subsection*{Определение классов}
\addcontentsline{toc}{subsection}{Определение классов}

\textbf{Класс Organization (organization.h / organization.cpp):}
\begin{itemize}
	\item Члены данных: \texttt{name} (string), \texttt{inn} (string), \texttt{employees} (int).
	\item Чистые виртуальные методы: \texttt{payTaxes()}, \texttt{report()}. Эти методы делают класс абстрактным.
	\item Конструкторы: конструктор по умолчанию, конструктор с параметрами, конструктор копирования.
	\item Общие методы: \texttt{hireEmployee()} (увеличивает количество сотрудников), \texttt{reklama()} (выводит рекламное сообщение).
\end{itemize}

\textbf{Класс Commersorg (commercial.h / commercial.cpp):}
\begin{itemize}
	\item Собственные члены данных: \texttt{revenue} (int), \texttt{taxRate} (int), \texttt{subscribers} (int), \texttt{socSeti} (vector<string>).
	\item Унаследованные члены данных: \texttt{name}, \texttt{inn}, \texttt{employees}.
	\item Конструкторы: конструктор по умолчанию, конструктор с параметрами (название, ИНН, сотрудники, выручка, ставка), конструктор с параметрами (включая подписчиков и соцсети), конструктор копирования.
	\item Переопределенные методы: \texttt{payTaxes()} (возвращает \texttt{revenue * taxRate / 100}), \texttt{report()} (формирует строку с информацией об организации, включая соцсети при наличии).
	\item Собственные методы: \texttt{expandBusiness()} (увеличивает количество подписчиков на 100), \texttt{distributeProfit()} (выводит бюджет для распределения).
\end{itemize}

\textbf{Класс Noncomersorg (noncomersorg.h / noncomersorg.cpp):}
\begin{itemize}
	\item Собственные члены данных: \texttt{purpose} (string), \texttt{foundingSource} (string).
	\item Унаследованные члены данных: \texttt{name}, \texttt{inn}, \texttt{employees}.

	\item Собственные методы: \texttt{conductProgram()} (выводит информацию о проводимой программе), \texttt{attractFunding()} (выводит информацию о привлечении финансирования и обновляет источник).
\end{itemize}

\subsection*{Разработка демонстрационного приложения}
\addcontentsline{toc}{subsection}{Разработка демонстрационного приложения}
Для демонстрации созданной структуры разработано консольное приложение (Clases.cpp).

\textbf{Демонстрация инкапсуляции:}

Данные классов (\texttt{revenue}, \texttt{purpose}) скрыты в секции \texttt{private}. Доступ к ним и изменение состояния объектов возможны только через публичные методы-геттеры и сеттеры.

\textbf{Демонстрация наследования:}

Объекты \texttt{Commersorg} и \texttt{Noncomersorg} содержат все поля и методы класса \texttt{Organization}. При вызове методов, унаследованных от базового класса, таких как \texttt{hireEmployee()}, объекты дочерних классов корректно используют свою реализацию.

\textbf{Демонстрация полиморфизма:}

Все объекты хранятся в векторе умных указателей на базовый класс \texttt{vector<unique\_ptr<Organization>>}. При вызове виртуальных методов (\texttt{payTaxes()}, \texttt{report()}) программа автоматически выполняет версию метода, соответствующую реальному классу объекта, благодаря механизму позднего связывания (динамическому полиморфизму).

В приложении продемонстрированы:
\begin{itemize}
	\item Создание объектов различных типов с использованием различных конструкторов.
	\item Использование \texttt{static\_cast} для вызова специфических методов дочерних классов.
	\item Хранение объектов в контейнере через указатели на базовый класс.
	\item Вызов виртуальных методов через указатели на базовый класс.
\end{itemize}
