# Функции в ООП (C++)

#ООП #cpp #функции

## 1. Перегрузка функций (Function Overloading)

**Перегрузка функций** — это возможность создавать в одной области видимости несколько функций с **одинаковым именем**, но с **разными наборами параметров** (сигнатурой).

Сигнатура функции определяется количеством, типами и порядком её параметров. **Возвращаемый тип в сигнатуру не входит.**

Это одна из форм **статического (compile-time) полиморфизма**, так как компилятор определяет, какую именно версию функции вызвать, ещё на этапе компиляции.

### Процесс выбора функции (Overload Resolution)

Когда вы вызываете перегруженную функцию, компилятор выполняет следующие шаги:
1.  Находит всех кандидатов с таким же именем.
2.  Выбирает из них те, для которых типы аргументов в вызове могут быть преобразованы в типы параметров функции.
3.  Определяет **наилучшее соответствие**. Если находится ровно одно наилучшее соответствие, используется оно. Если его нет или их несколько, компилятор выдаёт ошибку.

#### Пример:
```cpp
// 1. Версия для целых чисел
int sum(int a, int b) {
    return a + b;
}

// 2. Версия для чисел с плавающей точкой
double sum(double a, double b) {
    return a + b;
}

// --- Вызовы ---
int x = sum(3, 5);          // Компилятор выберет версию (1), т.к. аргументы - int
double y = sum(3.5, 6.0);   // Компилятор выберет версию (2), т.к. аргументы - double
````

> [!WARNING] Важное правило  
> Перегрузка функции **невозможна** только по возвращаемому типу. Это приведёт к ошибке компиляции, так как компилятор не сможет однозначно определить, какую функцию вызывать.
> 
> ```
> // ОШИБКА: переопределение функции
> int sum(int a, int b);
> float sum(int a, int b); // Нельзя! Сигнатура (int, int) уже занята.
> ```

## 2. Специальные методы класса

### Конструкторы (Constructors)

**Конструктор** — это специальный метод, который автоматически вызывается в момент создания объекта класса. Его основная задача — **инициализация** объекта, то есть приведение его в корректное начальное состояние.

#### Правила и особенности конструкторов:

1. **Имя** конструктора всегда совпадает с именем класса.
    
2. Конструктор **не возвращает значение**, даже void.
    
3. **Конструктор по умолчанию** (без аргументов) создаётся компилятором автоматически, но только если в классе не определено ни одного другого конструктора.
    
4. Конструкторы **не наследуются**. Дочерний класс вызывает конструктор родительского класса, но не получает его в своё распоряжение.
    
5. Параметры конструктора могут иметь **значения по умолчанию**, например MyClass(int x, int y = 10).
    
6. Конструкторы **не могут быть виртуальными** (virtual). Виртуальный механизм работает с уже созданными объектами, а конструктор только начинает их создавать.
    
7. Объект класса с нетривиальным конструктором нельзя включать в union (объединение).
    

#### Виды конструкторов:

- **Конструктор по умолчанию:** MyClass();
    
- **Конструктор с параметрами:** MyClass(int x, double y);
    
- **Конструктор копирования:** MyClass(const MyClass& other); — создаёт копию существующего объекта.
    
- **Конструктор перемещения (C++11):** MyClass(MyClass&& other); — "крадёт" ресурсы у временного объекта.
    

### Деструкторы (Destructors)

**Деструктор** — это специальный метод, который автоматически вызывается перед уничтожением объекта. Его основная задача — **освобождение ресурсов**, захваченных объектом за время его жизни (память, файлы, сетевые соединения и т.д.).

#### Правила и особенности деструкторов:

1. Имя деструктора — это ~ + имя класса (например, ~MyClass()).
    
2. Деструктор **не возвращает значение** и **не принимает параметров**.
    
3. В классе может быть **только один** деструктор.
    
4. Деструктор **не наследуется**. При уничтожении объекта дочернего класса автоматически вызывается сначала его деструктор, а затем деструктор родительского класса.
    
5. Деструктор **может (и часто должен) быть виртуальным**.
    
>[!IMPORTANT] Виртуальный деструктор  
> Если вы планируете работать с объектами дочерних классов через указатель на базовый класс, деструктор базового класса **обязательно** должен быть виртуальным. Иначе при delete base_ptr; вызовется только деструктор базового класса, что приведёт к утечке ресурсов дочернего.
    
6. Адрес деструктора нельзя получить.
    
7. Деструктор генерируется компилятором автоматически, если не определён пользователем.
    
8. Вызывается автоматически при выходе объекта из области видимости или при вызове оператора delete для объекта, созданного в динамической памяти. (Не free, free — это функция из C для памяти, выделенной malloc).

## 3. Правило трёх / пяти / нуля

Это набор правил C++, который помогает правильно управлять ресурсами (например, динамической памятью).

> [!NOTE] "Приличный программист при работе с динамикой должен..."  
> Если ваш класс вручную управляет ресурсом (например, содержит "сырой" указатель и выделяет для него память через new), то для корректной работы ему необходимо определить "большую тройку":

1. **Деструктор** — для освобождения ресурса.
    
2. **Конструктор копирования** — для глубокого копирования ресурса.
    
3. **Перегруженный оператор присваивания** (operator=) — для корректного присваивания с глубоким копированием.
    

> С появлением C++11 это правило расширилось до **"Правила пяти"**, добавив **конструктор перемещения** и **оператор присваивания с перемещением**.
> 
> Современный идеал — **"Правило нуля"**: не управляйте ресурсами вручную. Используйте "умные" указатели (std::unique_ptr, std::shared_ptr) и стандартные контейнеры (например, std::vector), которые делают всю грязную работу за вас.

## 4. Дружественные функции и классы (Friend)

**Дружественная функция** (friend) — это функция, которая не является членом класса, но имеет доступ к его private и protected членам.

Ключевое слово friend используется внутри определения класса, чтобы "дать разрешение" внешней функции или другому классу.

#### Пример:


```
class Vector2D {
private:
    double x, y;

public:
    Vector2D(double x_val, double y_val) : x(x_val), y(y_val) {}

    // Объявляем, что функция printVector является другом этого класса
    friend void printVector(const Vector2D& v);
};

// Сама функция не является методом класса Vector2D,
// но теперь может обращаться к v.x и v.y
void printVector(const Vector2D& v) {
    std::cout << "Vector(" << v.x << ", " << v.y << ")" << std::endl;
}
```

> **Когда это полезно?** Дружественные функции часто используются для перегрузки операторов, которые не могут быть членами класса (например, оператор вывода << для std::ostream).
