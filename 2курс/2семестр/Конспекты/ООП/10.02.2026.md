# Парадигмы программирования

Парадигма программирования — это совокупность идей и понятий, определяющих стиль написания компьютерных программ. Это высокоуровневый способ мышления о том, как структурировать и выполнять код.

## Виды парадигм

- **Императивное программирование** - Описывает процесс вычислений в виде последовательности инструкций, изменяющих состояние программы.
    
    - **Структурное программирование** - подвид императивного, использующий управляющие конструкции (циклы, ветвления, подпрограммы) вместо goto.
        
    - **[[Объектно-ориентированное программирование]] (ООП)** - программа представляется как совокупность взаимодействующих объектов.
        
    - **Автоматное программирование** - программа описывается как модель конечного автомата, переходящего из одного состояния в другое.
        
- **Декларативное программирование** - Описывает что нужно получить, а не как это сделать.
    
    - **Функциональное программирование** - программа состоит из чистых функций, без побочных эффектов и изменяемого состояния.
        
    - **Логическое программирование** - программа состоит из набора фактов и правил логического вывода (например, Prolog).
        
- **Событийно-ориентированное программирование** - выполнение программы определяется событиями (действия пользователя, сообщения от других программ, системные события).

****## Объектно-ориентированное программирование (ООП)

> **ООП** — это методология программирования, основанная на представлении программы в виде совокупности объектов, каждый из которых является экземпляром определённого класса, а классы образуют иерархию наследования.

### Основные понятия

#### Объект

Объект — это ключевое понятие в ООП. Это сущность, обладающая тремя характеристиками:

- **Состояние (State)**: Набор данных (атрибутов, полей), которые хранит объект в конкретный момент времени. Например, у объекта "Автомобиль" состояние — это цвет, скорость, уровень топлива.
    
- **Поведение (Behavior)**: Действия, которые объект может выполнять. Реализуется через методы (функции-члены). Поведение изменяет состояние объекта или взаимодействует с другими объектами. Например, "Автомобиль" может "ускоряться()", "тормозить()", "перекрашиваться()".
    
- **Уникальность (Identity)**: Каждый объект уникален и существует независимо от других, даже если их состояния идентичны. В памяти у каждого объекта свой адрес.
    

#### Класс

**Класс** — это "чертёж" или шаблон для создания объектов. Он описывает общую структуру (набор полей) и общее поведение (набор методов) для множества однотипных объектов.

- Например, класс Car описывает, что у всех автомобилей есть цвет и скорость, и все они могут ехать. А конкретные машины (myTesla, myLada) — это **экземпляры (объекты)** этого класса.
    

### Элементы объектной модели (Столпы ООП)

#### 1. Абстракция

**Абстракция** — это выделение значимых характеристик объекта и отбрасывание несущественных. Мы создаём упрощённую модель реальной сущности, которая важна для решения конкретной задачи.

> Пример: Моделируя объект User для сайта, нас интересуют его login, password, email. Нам не важны его рост, цвет глаз или любимая еда. Мы абстрагировались от ненужных деталей.

#### 2. Инкапсуляция

**Инкапсуляция** — это объединение данных (состояния) и методов их обработки (поведения) внутри одного объекта (класса) с сокрытием деталей реализации от внешнего мира. Доступ к данным предоставляется через открытый интерфейс (публичные методы).

- **Цель:** Защитить данные от прямого неконтролируемого изменения. Объект сам отвечает за целостность своего состояния.
    

> Пример: Мы не можем напрямую изменить скорость автомобиля. Мы вызываем метод accelerate(), а уже он внутри себя корректно изменяет приватное поле speed, возможно, с учётом уровня топлива или состояния двигателя. Мы не знаем, как именно он это делает, мы просто используем "педаль газа".

#### 3. Иерархия (Наследование)

**Иерархия** — это ранжирование или упорядочивание абстракций. В ООП ключевой формой иерархии является **наследование**.  
**Наследование** — это механизм, который позволяет создавать новый класс (потомок, дочерний класс) на основе уже существующего (предок, родительский класс).

- **Преимущества:**
    
    - **Переиспользование кода:** Потомок автоматически получает все поля и методы родителя.
        
    - **Построение иерархий:** Создание отношений "является" (is-a). Например, Dog **является** Animal.
        

> Пример: Класс Manager наследуется от класса Employee. Manager автоматически получает поля name, salary и методы work() от Employee, но может добавить свои специфичные, например, giveBonus().

#### 4. Полиморфизм

**Полиморфизм** (от греч. "много форм") — это способность объектов с одинаковым интерфейсом (набором методов) иметь разную реализацию этих методов.

- **Цель:** Работать с объектами разных дочерних классов через единый интерфейс родительского класса.
    

> Пример: У нас есть родительский класс Shape с методом draw(). Его наследники Circle и Square по-своему реализуют этот метод. Мы можем создать массив объектов Shape и для каждого вызвать draw(), не задумываясь, что именно мы рисуем — круг или квадрат. Программа сама выберет нужную реализацию в зависимости от реального типа объекта.

### Дополнительные концепции

- **Модульность** — это свойство системы, связанное с возможностью её декомпозиции на ряд внутренне связанных, но слабо зависимых между собой модулей. В ООП классы и группы связанных классов являются естественными модулями.
    
- **Контроль типов (Типизация)** — это механизм, который гарантирует, что операции выполняются над подходящими для них типами данных. В контексте ООП это означает, что нельзя, например, вызвать метод bark() ("лаять") у объекта класса Car.
    
    - **Важное замечание:** Ваше определение "Выдать родительский тип за дочерний" неверно. Наоборот, полиморфизм позволяет **выдавать дочерний тип за родительский** (объект Dog можно поместить в переменную типа Animal). Обратное преобразование (из Animal в Dog) требует явного и небезопасного приведения типов.
        
    - Про "существование объекта после разрушения породившего" — это скорее относится к управлению памятью и жизненным циклом объектов (например, композиция vs агрегация, или умные указатели), а не к контролю типов.

### Реализация в C-подобных языках (на примере C++)

В C++ и подобных языках класс — это сложный структурный тип данных, состоящий из:

1. **Членов-данных (Fields, Properties)**: Переменные, хранящие состояние объекта.
    
2. **Членов-функций (Methods)**: Функции, определяющие поведение объекта.
    

#### Области видимости (Модификаторы доступа)

Определяют, откуда можно получить доступ к членам класса. Являются ключевым инструментом **инкапсуляции**.

- public (Публичная область): Члены доступны из любого места программы, где виден объект класса. Это **открытый интерфейс** класса.
    
- protected (Защищенная область): Члены доступны внутри самого класса и внутри его классов-наследников.
    
- private (Приватная область): Члены доступны **только** внутри методов самого класса. Это скрытая реализация.
    

> В C++ по умолчанию все члены класса имеют модификатор private.

#### Специальные методы класса

- **Конструктор** — специальный метод, который вызывается автоматически при создании объекта класса (По умолчанию в порядке определения полей устанавливаются). Его задача — инициализировать начальное состояние объекта (присвоить значения полям).
- **Делегирующий конструктор** — https://learn.microsoft.com/ru-ru/cpp/cpp/delegating-constructors?view=msvc-170

#### Getters и Setters

Это пара методов для реализации контролируемого доступа к приватным полям.

- **Getter** (аксессор): Публичный метод, который **возвращает** значение приватного поля. Обычно имеет имя getX() или get_x().
    
- **Setter** (мутатор): Публичный метод, который **устанавливает** новое значение для приватного поля. Может содержать логику проверки (валидации) нового значения. Обычно имеет имя setX() или set_x().
    

```
class Car {
private:
    int speed; // Приватное поле

public:
    // Getter
    int getSpeed() {
        return speed;
    }

    // Setter с валидацией
    void setSpeed(int newSpeed) {
        if (newSpeed >= 0 && newSpeed <= 200) {
            speed = newSpeed;
        }
    }
};
```

#### Скрытый указатель this

Внутри нестатического метода каждого класса неявно существует указатель с именем this.


- **Что это?** *this* — это указатель на **текущий экземпляр объекта**, для которого был вызван метод.
    
- **Зачем нужен?**
    
    1. Для различения полей класса и локальных переменных/параметров с одинаковыми именами.
        
    2. Для возврата ссылки на текущий объект из метода (return *this*;), что позволяет выстраивать цепочки вызовов.
        

> Выражение "Напрямую к указателю не обращаемся, поэтому называется скрытым. Константным..." означает, что компилятор передает этот указатель в метод неявно (скрыто). В большинстве случаев он является константным указателем (Car *const this), то есть сам указатель изменить нельзя (он всегда будет указывать на текущий объект), но данные, на которые он указывает, изменять можно.

Можно хитро использовать, возвращать как результат функции.
